# DavyAgent — Pending Work Items
# Date: 04/16/2026

## 1. Add skill/tool package auto-discovery via entry points
No built-in registry or discovery mechanism exists. Third-party tools must be
manually configured in `config/default.toml`. Add Python `entry_points`-based
auto-discovery so installed packages can register tools automatically.

## 2. Add MCP Python 3.11 compatibility fallback
MCP support silently returns an empty list on Python 3.11
(`davyagent/tools/registry.py:29`) because the `openai-agents` SDK requires
Python 3.12+. Add a clear warning or error, and evaluate an alternative MCP
client that supports Python 3.11.

## 3. Remove Ctrl+O, Ctrl+P, Ctrl+N key bindings from davy_cli.py
Remove the Ctrl+O, Ctrl+P, and Ctrl+N keyboard shortcut bindings from
`davy_cli.py`. The demo functions behind those bindings must remain in place
for debugging purposes.

## 4. Implement plan/execution modes with Ctrl+P binding and /mode command
No execution modes exist in the library layer — only LLM persona roles and TUI
lifecycle states. Add two modes:
- plan — agent reasons and drafts without executing actions
- execution — agent runs tools and actions

Modes must be switchable via:
- Ctrl+P key binding
- `/mode` command

## 5. Fix stack dump shown when listing providers in /setup command
When the providers listing option is selected inside the `/setup` command flow,
a stack trace is displayed instead of the provider list. Investigate and fix
the root cause in `davy_cli.py`.

## 6. Show "Done. DavyAgent is ready" notification after provider/model changes
After any of the following events, append a notification frame in the scrollable
window with the message "Done. DavyAgent is ready":
- Provider change
- Model change
- Listing of available models or providers

## 7. Add the command "/role" to display and change roles
The command /role should launch a menu in the scrollable window, and the
available roles can be selected. Once a new role was selected, the
DavyAgent notification from #6 must appear.
