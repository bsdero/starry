# Plan: Rename everything to "Starry" (StarryLib / StarryCLI)

**Goal:** Replace every `davy*` / `Davy*` / `DAVY*` brand identifier with
the new **Starry** brand.  The two public-facing names are:

| Old brand | New brand | Context |
|---|---|---|
| `DavyAgent` / `davyagent` | `StarryLib` / `starrylib` | the library package |
| `DavyCLI` / `davy_cli` | `StarryCLI` / `starry_cli` | the TUI / CLI |

`old_plans_already_applied/`, `.git/`, and `.venv/` are **out of scope**.
`certs/davy.labs.lenovo.com.crt` (and the copy inside `davyagent/`) keep
their name — it is a server domain, not a brand identifier.

## Execution order

1. **Rename `davy_cli.py` → `starry_cli.py`** (top-level file).
2. **Rename directory `davyagent/` → `starrylib/`** (package root).
3. **Update `pyproject.toml`** (package name, py-modules, include glob,
   console script, entry-point group).
4. **Bulk search-and-replace `davyagent` → `starrylib`** across all `.py`,
   `.toml`, `.md`, `.sh`, `.json` files (excluding `old_plans_already_applied/`,
   `.git/`, `.venv/`).
5. **Update display strings** (`DavyAgent` → `StarryLib`, `DavyCLI` →
   `StarryCLI`, `DAVYCLI` → `STARRYCLI`) in `starry_cli.py`,
   `config/default.toml`, docs, `install.sh`.
6. **Update env var names** (`DAVY_API_KEY` → `STARRY_API_KEY`,
   `DAVYAGENT_SESSIONS_DIR` → `STARRYLIB_SESSIONS_DIR`).
7. **Update home-dir paths** in `settings.py`, `store.py`, `agent_store.py`,
   `todowrite.py`, `starry_cli.py`; rename the `_DAVYAGENT_DIR` constant.
8. **Update provider name** `"davy"` → `"lenovo"` in `default.toml` and
   `settings.py` (both the key and the default fallback strings).
9. **Delete `davyagent.egg-info/`** (stale after package rename).
10. **Re-install in dev mode**: `pip install -e .` to rebuild
    `starrylib.egg-info/` and register the `starrylib.tools` entry-point
    group.
11. **Run tests** (`pytest`) to confirm no import-path breakage.
12. **Update `AGENTS.md`**, `CLAUDE.md`, and `docs/`.
