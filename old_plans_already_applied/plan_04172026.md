# DavyAgent — Pending Work Items
# Date: 04/17/2026

## 1. Native Python tool implementations

Implement all agent tools as individual Python modules under
`davyagent/tools/implementations/`. Each module exposes a
`SCHEMA` dict (OpenAI function-calling format) and an
`execute(**kwargs) -> dict` function.

### Tools implemented

| Module          | Tool name   | Description                        |
|-----------------|-------------|------------------------------------|
| bash.py         | bash        | Execute shell commands             |
| read.py         | read        | Read files or list directories     |
| glob.py         | glob        | Find files by glob pattern         |
| grep.py         | grep        | Search file contents by regex      |
| edit.py         | edit        | Replace exact text in a file       |
| write.py        | write       | Create or overwrite a file         |
| task.py         | task        | Launch autonomous subagents        |
| webfetch.py     | webfetch    | Retrieve web content via HTTP      |
| todowrite.py    | todowrite   | Manage a persistent task list      |
| skill.py        | skill       | Load specialized skill files       |
| question.py     | question    | Request user input interactively   |

### Supporting files

- `davyagent/tools/tool_loader.py` — Maps modes to tool
  sets. Provides `get_tool_schemas(mode)` and
  `get_tool_executor(mode)`.
- `davyagent/tools/TOOLS.md` — Human-readable reference
  for all tools, args, return values, and mode availability.

## 2. Mode-aware tool delivery to LLM

The tool set sent to the LLM depends on the execution mode:

### Plan & Research mode
Tools: `todowrite`, `task`, `question`, `webfetch`,
`skill`, `glob`, `grep`, `read`

Read-only and research operations. No file writes or shell
execution.

### Execution & Implementation mode
All plan tools plus: `bash`, `edit`, `write`

Full read/write/run capabilities.

## 3. Session.mode and chat_auto()

Added to `davyagent/agents/session.py`:

- `session.mode` property — get/set current mode
  (`"plan"` or `"execution"`).
- `session.get_tool_schemas()` — returns schemas for mode.
- `session.get_tool_executor()` — returns executor map.
- `session.chat_auto(user_input)` — async generator that
  uses `chat()` when no tools are active, or
  `chat_with_tools()` with auto-selected schemas and
  executor when tools are configured.

`AgentPool.spawn()` accepts a `mode` parameter
(default `"execution"`).

## 4. TUI integration in davy_cli.py

`handle_ai_response()` updated to:
- Sync `_exec_mode` → `session.mode` before each call.
- Use `session.chat_auto()` instead of `session.chat()`.
- Handle new event types:
  - `tool_call` → warning frame showing tool name + args.
  - `tool_result` → inline notification with result preview.
  - `done` → render final response for non-streaming path.
