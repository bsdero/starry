## 1. question tool — TUI input collection

When the LLM calls the `question` tool the TUI must
pause and collect real user input before resuming.

### Current state
`question.execute()` returns a `user_input_required`
stub dict. The TUI renders a warning frame but does
not collect input; the tool loop continues immediately
with the empty stub as the tool result.

### Required work
- In `handle_ai_response()`, detect `tool_result`
  events where `name == "question"` and the result
  contains `type: "user_input_required"`.
- Pause the event loop using `asyncio.Queue`: the
  coroutine `await`s a queue that the TUI input
  handler writes to when the user submits a reply.
- Show a dedicated inline prompt frame (distinct from
  the normal chat input) labelled with each question.
- On user submission, inject the answer(s) back as the
  tool result string and resume the `chat_with_tools()`
  loop.
- Multiple questions in one call: collect all answers
  before resuming.

## 2. task tool — real subagent dispatch via AgentPool

When the LLM calls the `task` tool, a real child
Session must be spawned, the subtask executed, and
the result returned to the parent session.

### Current state
`task.execute()` returns a `subagent_request` stub.
No child session is spawned.

### Required work
- Expose the active `AgentPool` to `task.execute()`
  via module-level reference or context var injected
  at startup.
- Add `AgentPool.run_subtask(prompt, role, mode)`
  method: spawns a child Session, calls
  `session.chat_complete()`, terminates the child,
  returns the response string.
- Wire `task.execute()` to call `run_subtask()`.
- Surface the subagent response to the parent TUI
  as an inline notification or frame.

## 3. Tool result streaming

`chat_with_tools()` is non-streaming. After the tool
loop completes, the final LLM answer is a single
`done` event with no preceding `token` events.

### Required work
- After all tool results are appended to `messages`,
  replace the non-streaming `create()` call with
  `create(..., stream=True)`.
- Yield individual `token` events as they arrive.
- Yield the final `done` event with the assembled
  text.
- Update conversation history only after the stream
  completes.
- TUI already handles `token` events for progressive
  frame rendering — no TUI changes needed.

## 4. Tool confirmation in plan mode

In `plan` mode, tool calls should require explicit
user approval before execution (skip-and-continue
if denied).

### Required work
- Add `require_confirmation: bool` to `Session`
  (default `False`; auto-set `True` when
  `mode == "plan"`).
- In `chat_with_tools()`, before dispatching each
  tool call, yield a `tool_confirm_request` event
  carrying the tool name and args.
- Pause via `asyncio.Queue`; resume when the caller
  sends an approval boolean.
- If the user denies: skip execution, inject an
  empty/error result, continue the loop (do not
  abort the whole turn).
- TUI: show an inline yes/no prompt on
  `tool_confirm_request` events.

## 7. --session restore: auto-switch provider/model

When a session is restored with `--session <hash>`,
the provider and model active at save time are not
restored — the CLI starts with its default provider
and model instead.

### Current state
`store.save()` already persists `"provider"` and
`"model"` keys in the session JSON.
The `--session` restore path loads history and
replays display_log but does not call
`switch_provider()` or update `_cur_model`.

### Required work
- In the `--session` restore block of `main()` (and
  in the `/sessions` in-process restore path):
  1. Read `data["provider"]` and `data["model"]`
     from the loaded JSON.
  2. If `provider != _cur_provider`, call
     `_da_session.switch_provider(provider,
     _da_settings)` and update `_cur_provider`.
  3. Set `_da_session._agent.model = model` and
     update `_cur_model`.
  4. Update `_avail_models` if the model is not
     already in the cached list for that provider.
- Show a notification after restore:
  `"Session restored. Provider: {p}, Model: {m}"`
  with icon `"↩"`.
- Handle the case where the saved provider is no
  longer configured in settings (warn, fall back to
  default, continue loading history).
